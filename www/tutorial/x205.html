2.2. Conversion Units

For a variety of reasons, there is no one standard way of representing an image. 8-bit grayscale, 24-bit RGB, YUV 4:2:2, Bayer, grayscale, all of these are fairly common image formats. Frequently, however, an image processing algorithm can only deal with images in a very specific colorspace and binary representation.

libcam provides a set of conversion CamUnit classes to handle this. These units can be found in the "convert" namespace. Each unit can only perform a certain number of conversions, and also has certain input requirements. A few examples of some of the conversion units and some the conversions they can perform are:

convert.fast_debayer

  • Bayer RGGB -> BGRA

  • Bayer GBRG -> BGRA

  • Bayer GRBG -> BGRA

  • Bayer BGGR -> BGRA

convert.colorspace

  • 8bpp Gray -> 24bpp RGB

  • 8bpp Gray -> 32bpp RGBA

  • YUV 4:2:0 Planar-> 24bpp RGB

  • YUV 4:2:2 -> 24bpp RGB

  • and more...

convert.jpeg

  • 24bpp RGB -> JPEG

  • JPEG -> 24bpp RGB

  • 8bpp Gray -> JPEG

  • and more...

To use one, add the desired unit to the end of a CamUnitChain and specify the desired output format. For example, as we saw in the previous chapter, the example input unit produces 24bpp RGB images. To create a chain that compresses these to JPEG, we could do the following: 
    CamUnitChain *chain = cam_unit_chain_new ();

    CamUnit *example_input = 
        cam_unit_chain_add_unit_by_id (chain, "input.example");

    CamUnit *jpeg_unit = 
        cam_unit_chain_add_unit_by_id (chain, "convert.jpeg");

    cam_unit_set_preferred_format (jpeg_unit, CAM_PIXEL_FORMAT_MJPEG, 0, 0);

If we then attach a callback function to the chain's "frame-ready" signal, then the image data received in that function should be JPEG compressed.
